### NemoClaw VM rebuild (final run)
- Switched from i3 to XFCE — i3 was too steep a learning curve
- VNC section removed from Tart guide — security concern, access via Tart window instead
- `dbus-x11` added to XFCE install — required for XFCE to work properly
- `xvfb` must be installed separately — not included by default, causes "Cannot open display" without it
- XFCE auto-starts on boot via Tart window, no VNC needed
- NODE_OPTIONS fix goes in **sandbox** `~/.bashrc` not host VM
- Launchd plist set up for auto-start — `KeepAlive` removed so VM can be stopped cleanly
- NemoClaw install ran cleanly: sandbox named `mira-sandbox`, policies: pypi, npm, brave, discord, telegram

### Workspace migration
- tar.gz backed up to Google Drive (`Mira's OpenClaw Backups` folder, dated 2026-04-04)
- `openshell sandbox upload` is the correct transfer method into sandbox
- Full command: `openshell sandbox upload mira-sandbox /path/to/file.tar.gz /sandbox/home/sandbox/filename.tar.gz`
- Workspace landed in `.openclaw` but symlinks point to `.openclaw-data/workspace/` — need `--strip-components=1` to extract to right place
- **STATUS: workspace files extracted, but new Mira showing BOOTSTRAP flow** — workspace path mismatch still being resolved

### Key technical discoveries
- NemoClaw sandbox has NO host filesystem access (`/host` doesn't exist)
- `nemoclaw exec` is not a valid command (valid: connect, status, logs, policy-add, policy-list, destroy)
- Google Drive curl blocked by sandbox policy
- Python HTTP server from host IS reachable from sandbox (same VM)
- Workspace symlink: `~/.openclaw/workspace` → `/sandbox/.openclaw-data/workspace`
- Correct extraction path: `cd /sandbox/.openclaw-data && tar -xzf ... --strip-components=1`
